# Recently deleted packages

The recently deleted packages page allows you to view deleted packages that are pending permanent deletion. You can also restore to the repository and permanently delete packages from this page. This can be useful in cases of accidental artifact deletion. 

## Package retention

Deleted packages are retained for a 2 week period and then are automatically deleted.

## Restore packages

To restore an individual package, click on the arrow icon on the right hand side of the table row. If you want to restore multiple packages at once, select your packages then choose 'Restore' from the actions menu on the bottom of the page. 

These packages will be restored to their original repository. 

<BlockImage src={img6fe0d48_list_tags} alt="List tags"></BlockImage>

## Permanently delete packages

To permanently delete an individual package, click on the trash icon on the right hand side of the table row. If you want to permanently delete multiple packages at once, select your packages then choose 'Permanent delete' from the actions menu on the bottom of the page. 

**Note:** once permanently deleted, this packages can no longer be restored. 

<BlockImage src={img6fe0d48_list_tags} alt="List tags"></BlockImage>

# Deleted repositories

When a repository is deleted, all of the packages will be transferred to recently deleted packages. When you restore a package from a deleted repository, the repository will be automatically restored too.
